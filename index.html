form

Управляющие формой атрибуты:

  1. action
  2. method
  3. enctype
  4. novalidate - о нем чуть позже

form action

action - обязательный, отвечает за url, на который послать информацию из формы

form method

method - метод запроса

Доступны только 2: "get" и "post"

form enctype

enctype - способ кодирования информации в форме

  1. application/x-www-form-urlencoded - по умолчанию
  2. multipart/form-data - для файлов только такой
  3. text/plain

UX

Accessibility форм

Нужно, чтобы:

  • На мобильном устройстве были нативные (или хорошо подстроенные свои) элементы управления интерфейсом
  • Контролы интерфейса можно было выбрать с клавиатуры (табуляция), пройти по ним и активировать их пробелом и нажатием клавиши enter
  • Ко всем input был соответствующий label или хотя бы title

Как стилизовать элементы формы?

Свести сложные элементы к как можно более нативным.

  • desktop - кастомизируем, как хотим
  • touch - дать выбрать через нативные контролы

Валидация

Дизайнеры не любят нативную валидацию. И есть, за что.

Валидация

Нужно помогать пользователям изначально вводить правильно!

  1. Показать правильный контрол (!)
  2. Провалидировать поля со стороны JS через Constraint Validation API
  3. Если ошибки ввода есть, то показать сразу как поле потеряло фокус (onblur)

form novalidate

novalidate - атрибут формы, который позволяет отправлять форму, даже если поля в ней заполнены неправильно.

При том все селекторы псевдоклассов, JS API работают!
